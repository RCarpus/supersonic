Supersonic Ear Training API Documentation

Author: Ryan Carpus

This API provides an interface with the Supersonic ear training app to handle user login/registration and store and retrieve saved user data.

Navigation

  • Register a new user
  • Login
  • Update a user's basic info and settings
  • Delete an existing user
  • Retrieve data for a specific user
  • Add a new training record
  • Delete a custom training session and all of it's records
Function Endpoint URL HTTP Method Request body data format Response body data format

Register a new user

back to top 		/users/register POST A JSON object holding data for a new user
example:
{
  "Username": "johndoe"
  "Password": "aStrongPasSwOOrd"
  "Email": "johndoe@gmail.com",
}
 A JSON object holding data for the new user.
example:
{
  "_id" : "619b8cfc843f95b8e96b8e66",
  "Username": "johndoe"
  "Password": "$2b$10$FXkr5zcaRCIgY1icGUFWvY9MQlgQQ/ei"
  "Email": "johndoe@gmail.com",
  "Settings" : {
    "NoteDuration" : "1000",
    "SoundWaveType" : "sine"
  },
  "Stats": []
}

}

Update a user's basic info and settings

back to top 		/users/[Username] PUT A JSON object holding the data to be updated for the user. Email, Password, and Settings are all optional fields which may be included.
example:
{
"email" : "KILLALLHUMANS@robots.com"
"Password": "IhateHumaNs"
} 		A JSON object holding the complete updated user data.
example:
{

"Username" : "KillerRobot",
"Password" : "$2b$10$FXkr5zcaRCIgY1icGUrESuPq2WzfRXdY1.oISy9FWvY9MQlgQQ/ei",
"Email" : "KILLALLHUMANS@robots.com",
"Data" : "l;aksdfa"
}

Delete an existing user

back to top 		/users/deregister/[Username] DELETE None A text message indicating that a user was deleted
example:
"KillerRobot was deleted."

Retrieve data for a specific user

back to top 		/users/[Username] GET None A JSON object holding all data for the specified user.
